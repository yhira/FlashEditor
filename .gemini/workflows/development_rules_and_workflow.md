# FlashEditor 開発ルールとワークフロー

本プロジェクトにおいて、AIアシスタントや開発者が作業を進める際は、以下の厳格なルールとワークフローに従うこと。

## 1. コミットに関する絶対ルール
- **【厳守】勝手にコミットを実行しない**: 
  AIアシスタントは、いかなる場合であってもユーザーの明示的な許可・指示がない限り、自動で `git commit` を実行してはならない。
  変更が完了した際は必ず対象ファイルとコミットメッセージの案を提示し、ユーザーの承認を得てからコミットを行うこと。

## 2. ドキュメント更新のワークフロー
- **機能追加・修正時の README 更新の義務**: 
  プログラムの機能追加、仕様変更、または機能に関するバグ修正を行った場合は、原則として**必ずその都度 `README.md` の記述内容を確認し、実際の仕様と一致するように修正・更新する手順をワークフローに組み込むこと**。
- 過去に予定されていた未実装の仕様や、現在使われていない独自の仕様説明が残らないよう、コードの実態とドキュメントを常に同期させること。
