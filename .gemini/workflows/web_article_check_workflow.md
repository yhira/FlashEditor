# Web記事整合性チェックワークフロー

ユーザーから「Web記事の整合性をチェックして」といった趣旨の指示があった場合、AIアシスタントは以下のワークフローに従って処理を実行すること。

## 対象記事の一覧

以下の3つの記事を順番にチェックする対象とする。

1. https://nelab.jp/66
2. https://nelab.jp/85
3. https://nelab.jp/75

## 実行ワークフロー

1. **情報の取得**: `read_url_content` 等の利用可能なツールを使用して、上記3つのURLのコンテンツ（テキスト）をそれぞれ読み込む。
2. **整合性のチェック・精査**: 読み込んだ記事の内容について、以下の観点から精査を行う。
   - 記事の内容に矛盾や整合性の取れていない箇所がないか。
   - 更新漏れと思われる情報や、文脈がおかしい部分がないか。
3. **修正箇所の指摘**: チェックが完了したら、修正が必要と思われる箇所をユーザーに報告する。
   - 問題がない場合は「特に修正箇所は見当たりませんでした」と報告する。
   - 修正箇所がある場合は、「どの記事の」「どの部分が」「どのように問題か（理由）」を明確にし、可能な限り「修正案」を日本語で分かりやすく提示すること。
4. **修正版マークダウンの出力**: ユーザーから「変更がある（修正が必要な）記事のマークダウンを出力して」と指示された場合、修正を反映させた記事全文のマークダウンデータを、**Artifacts（アーティファクト）機能を用いて新たなマークダウンファイルとして出力**すること。
